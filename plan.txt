EACH MODULE MUST IMPLEMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

loader.py
  - load_pdf_from_path(file_path) → List[Document]
  - load_pdf_from_bytes(bytes, filename) → List[Document]  ← for Streamlit
  - get_document_stats(documents) → dict  ← pages, chars, avg chars/page

chunker.py
  - split_documents(documents, chunk_size, chunk_overlap) → List[Document]
  - Use separators: ["\n\n", "\n", ". ", " ", ""]
  - Tag every chunk with chunk_index in metadata
  - get_chunk_stats(chunks) → dict

embeddings.py
  - get_embedding_model() → LangChain Embeddings object
  - Support "openai" and "huggingface" via EMBEDDING_PROVIDER in config
  - Clear error messages for missing API keys

retriever.py
  - build_vector_store(chunks, embeddings) → VectorStore
  - get_retriever(vector_store, top_k) → Retriever
  - Support FAISS and Chroma switchable via VECTOR_STORE in config
  - save_vector_store() and load_vector_store() for persistence

llm.py
  - get_llm() → LangChain BaseLLM
  - Support "openai" and "gemini" via LLM_PROVIDER in config
  - Expose temperature, model name via config

pipeline.py
  - build_rag_chain(retriever, llm) → Runnable chain
  - Use a clear prompt template with {context} and {question} variables
  - Format retrieved source chunks for display
  - ask(chain, question) → {"answer": str, "source_chunks": List[Document]}

app.py  ← Streamlit UI
  - Sidebar: PDF uploader + processing status + chunk/page stats
  - Main area: chat interface with message history
  - Show source chunks used for each answer (expandable)
  - Session state management (vector store persists across questions)
  - Loading spinners during indexing and inference
  - Provider selector (OpenAI vs Gemini) in sidebar

1. MULTI-PDF SUPPORT

3. CONFIDENCE / RELEVANCE SCORING
   - Return cosine similarity score alongside each retrieved chunk
   - Display a relevance badge in the UI (High / Medium / Low)

6. STREAMING LLM RESPONSES
   - Stream tokens to Streamlit in real time (st.write_stream)
   - Do not wait for full response before displaying

7. EXPORT CHAT HISTORY
   - Button in sidebar to download full Q&A as a .txt or .json file

8. VECTOR STORE PERSISTENCE
   - Save FAISS index to disk after indexing
   - Load on startup if index already exists for the uploaded file
